# Uso de Markdown
## 1. ¿Qué es Markdown?

Markdown es un lenguaje de marcado ligero que permite dar formato a texto utilizando una sintaxis simple y fácil de leer. Fue creado por John Gruber con el objetivo de hacer que los documentos sean legibles en su forma sin procesar y al mismo tiempo convertibles a HTML de manera sencilla.

## ¿Por qué se utiliza en proyectos de software?

Para documentar repositorios en GitHub, GitLab o Bitbucket.

Para crear archivos README claros y organizados.

Para escribir documentación técnica sin depender de editores complejos.

Para mantener un formato compatible entre múltiples plataformas.

Porque es ligero, rápido y fácil de aprender.

- Elemento 3

## 3. Ventajas de usar Markdown con GitHub

Compatibilidad nativa: GitHub interpreta automáticamente los archivos .md.

Control de versiones: permite rastrear cambios en la documentación igual que en el código.

Visualización clara: los archivos se muestran formateados directamente en el repositorio.

Estandarización: facilita la colaboración entre equipos sin depender de formatos como Word.

Ligero y portable: funciona en cualquier sistema operativo y editor de texto.

Ideal para documentación continua: perfecto para proyectos que se actualizan frecuentemente.
